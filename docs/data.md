# Data Directory Structure

The repository tracks only the directory skeleton for `data/`. Real clinical data, generated outputs, logs, metrics, and local debug artifacts are ignored by Git.

## Expected Structure

```text
data/
├── README.md
├── inputs/
│   ├── CARMEN1_mappings.tsv
│   └── txt/
│       ├── ann/
│       │   └── *.ann
│       ├── raw/
│       │   └── *.txt
│       ├── masked/
│       │   └── *.txt
│       └── identify/
│           └── *.txt
└── outputs/
    ├── debug/
    │   └── logs/
    ├── logs/
    └── metrics/
```

## Inputs

- `data/inputs/CARMEN1_mappings.tsv` maps CARMEN document identifiers to their metadata.
- `data/inputs/txt/raw/` contains the original text files.
- `data/inputs/txt/ann/` contains the annotation files with the same base names as the raw text files.
- `data/inputs/txt/masked/` contains masked text files with entity type placeholders.
- `data/inputs/txt/identify/` contains text files where detected spans are wrapped with `[**...**]`.

When `identify/` is missing, the loader can create it from `raw/` and `ann/`. For benchmark runs that compare all formats, keep `raw/`, `masked/`, and `identify/` aligned by file name.

## Outputs

- `data/outputs/` stores model outputs such as `output_<model>.json`.
- `data/outputs/metrics/` stores per-model metrics and benchmark summaries.
- `data/outputs/debug/` stores debug artifacts generated by the pipeline.
- `data/outputs/debug/logs/` is the default log location in `.env.example`.
- `data/outputs/logs/` is available for deployments that configure `LOG_DIR=data/outputs/logs/`.

## Git Policy

The `.gitignore` keeps all data files ignored. Only `data/README.md` and `.gitkeep` files should be committed under `data/`.
